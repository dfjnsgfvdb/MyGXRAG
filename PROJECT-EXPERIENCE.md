# 项目实践说明

## 项目名称

遥感知识图谱可视化与 RAG 问答系统

## 项目背景

在遥感数据分析和领域知识管理场景中，图谱结构适合表达实体、关系和上下文信息。为了提升知识浏览和解释效率，本项目将知识图谱可视化与大模型问答结合，构建了一个可交互的知识图谱工作台。

该项目适合作为前端工程实践、实习项目复盘或简历项目说明，重点体现以下能力：

- Vue 3 + TypeScript 项目开发。
- 复杂可视化组件封装。
- 前后端接口联调。
- 大模型 API 接入与安全代理。
- 工程结构梳理和功能裁剪。

## 我的主要工作

- 负责前端主页面搭建，完成知识图谱导航、图谱画布和 RAG 对话区布局。
- 基于 Sigma.js 和 Graphology 实现图谱节点、边、图例、缩放和拖拽交互。
- 封装 `AIAssistant.vue` 对话组件，实现多轮对话、流式响应解析和 Markdown 展示。
- 设计后端代理接口，统一处理图谱 mock 数据和通义千问模型调用。
- 将大模型密钥从前端剥离，改为后端读取本地配置文件，降低密钥泄露风险。
- 对原项目进行功能收敛，删除与知识图谱无关的登录、数据集、影像处理、地图等模块。
- 补充项目 README、接口说明和组件文档，使项目结构和交付说明更清晰。

## 技术难点

### 1. 图谱渲染和交互状态管理

图谱节点和边数量较多时，渲染性能和交互稳定性容易受影响。项目通过 Graphology 管理图结构，通过 Sigma.js 负责渲染，并把图例筛选、节点高亮、布局切换等状态集中在组件内部维护。

### 2. 图谱上下文与大模型问答结合

RAG 对话不仅需要用户问题，还需要当前图谱的节点、边和关系上下文。前端通过组件引用获取当前图谱数据，后端将上下文组装进 system prompt，再调用 `qwen-plus-2025-07-28` 生成回答。

### 3. API Key 安全处理

大模型 API Key 不适合放在前端代码中。项目通过 `express.js` 提供后端代理，并从 `ai.config.json` 或环境变量读取密钥，前端只访问本地接口。
